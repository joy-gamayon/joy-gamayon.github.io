Problem Solving








It was hard for me to solve JavaScript problems. I felt overwhelmed trying to remember all the instructions and methods. I was trying to learn too much too fast, and it made my thoughts all mixed up.
I got stuck trying to figure out how to loop through certain parts of an array. I didn't know which method or code to use.
It took me a long time to solve each problem. I felt like I was just sitting at my computer, not making any progress.
To try and do better, I went back to the basics and looked over old exercises. I searched on Google for help with specific array methods. I also checked out resources like MDN and W3Schools.
I asked for help in Discord channels, like the help desk and after-hours support. I wanted to see if others were having the same problems.
I tried writing out my problems in simple steps using pseudocode, but it was tough to know where to start. Still, it helped me think clearly once I figured out the solution.
Throughout the exercise, I've been learning how to solve problems. I kept testing my code until I got the right answers. Writing pseudocode helped me understand each step and method I used.
As well as, rubberducking which helped me figure out simple problems with my code, especially wrong typos and using wrong syntax.










It was tough, but I managed to solve a problem elegantly for a few questions. It felt really good when I got the answers right without too much trouble.
I followed the instructions carefully and tackled each problem one step at a time.
For example, I created an empty value for a property in an object and made variables with values, then added them to an array.
It was smooth sailing, and I got the answers correct. Also, using the pseudocode made my methods clear and easy to understand.










During the JavaScript-kata activities, I've used different ways to solve problems.
I feel good about searching for help on Google. It helps me find solutions and examples. I'm also pretty good at explaining my problems to someone else, like my partner.
It often helps me understand things better.
Using pseudocode has been really helpful. It makes my code clearer and easier to understand. I feel confident using this method, and I'll keep using it.
Error messages are very helpful to use during the activites. I am quite confident utilizing it. It helps me to spot any errors in my coding and gives you the right instructions.
Using console.log still makes me unsure. I didn't use console.log much during the exercises. I preferred testing my code in the terminal. As I find easier to understand errors and right answers.
But, I will work on it and make use of the console.logging. I am aware it will be a great help for solving problems.
I'm also learning to ask for help from peers and coaches, even though it's still challenging for me.










Asking for help from peers and coaches is still a bit hard for me. I usually want to solve problems on my own and learn from my mistakes.
But I realize I get frustrated when I don't ask for help. I always end up wasting my time trying to figure out the solutions instead of just asking some help.
I'm afraid of asking simple questions or being judged. I know asking for help would make things easier,
so I need to work on having the courage to ask questions without worrying.
